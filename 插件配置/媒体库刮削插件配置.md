# 媒体库刮削插件配置

> 对已入库历史媒体补海报、NFO、演员等信息；**非日常下载主流程**，需要时手动运行一次。

## 前置条件

- 已完成 [Readme.md](../Readme.md) §9 `init-mpv2`，媒体已整理至 `media/` 目录。
- MoviePilot 插件市场已安装「媒体库刮削」插件。
- 明确本次要修复的媒体范围（建议先小范围测试）。

## 适用场景

| 场景 | 说明 |
|------|------|
| 历史媒体缺元数据 | 老资源入库时未刮削或刮削失败 |
| 补充海报/NFO | 缺封面、简介、演员列表等 |
| 修复识别错误 | TMDB 匹配错了，需重新关联 |
| 批量补全信息 | 迁移旧库、手工整理后的批量修复 |

## 不适用场景

| 场景 | 应使用的方案 |
|------|--------------|
| 日常订阅/搜索下载 | MoviePilot 自带整理已含自动刮削 |
| 手动放入待整理资源 | [目录实时监控插件配置.md](目录实时监控插件配置.md) |
| 刷流目录 | 不参与刮削，完全隔离 |
| 下载中间目录 | `downloads/media`、`downloads/manual` 尚未最终入库 |

> MoviePilot 正常整理流程已自动刮削，本插件主要用于**补刮削**和**历史修复**，不建议长期定时全库运行。

## 目录与隔离原则

### 只刮最终媒体库

```text
/volume1/media-data/media/真人电影
/volume1/media-data/media/真人剧集
/volume1/media-data/media/动漫电影
/volume1/media-data/media/动漫剧集
/volume1/media-data/media/综艺
/volume1/media-data/media/纪录片
/volume1/media-data/media/短剧
/volume1/media-data/media/小电影
```

私享影库按需添加：

```text
/volume1/media-data/media/私享影库
```

### 不要填写

| 路径 | 原因 |
|------|------|
| `downloads/media` | 下载中间目录，整理完成后文件会移走 |
| `downloads/manual` | 由目录实时监控处理，尚未最终入库 |
| `downloads/brush` | 刷流目录，不应生成 NFO 或进 Emby |
| `downloads/private` | 私密暂存，不在自动整理范围 |

## 配置步骤

### 1. 基本开关

| 选项 | 建议值 | 说明 |
|------|--------|------|
| 启用插件 | **默认关** | 需要修复时再开启 |
| 立即运行一次 | 需要时手动开 | 跑完即关 |
| 覆盖模式 | 不要覆盖已有元数据 | 避免覆盖已修正的中文标题、海报 |
| 执行周期 | 留空 | 不建议 cron 定时全库刮削 |

**仅在整库历史刮削完全错误时**，才考虑「强制覆盖所有元数据」。

### 2. 刮削路径

按上节「只刮最终媒体库」列表填写；路径过多时可先只填缺元数据最多的几个分类。

### 3. 排除路径

推荐默认填写：

```text
@eaDir
#recycle
.recycle
/volume1/media-data/media/私享影库
```

若私享影库也需要补刮削，删除最后一行即可。

## 为什么不建议定时运行

- 全库刮削 CPU 占用高，大库可能持续数小时。
- 频繁访问 TMDB，可能触发 API 限流。
- 反复刷新 Emby，影响观看体验。
- 可能覆盖人工修正过的元数据（即使选了「不覆盖」，部分字段仍可能变动）。

**推荐做法**：发现缺元数据 → 临时启用 → 手动运行一次 → 验证结果 → 关闭插件。

## 验证步骤

```text
1. 先对一个分类或小目录测试（如 media/短剧）
2. 覆盖模式选「不要覆盖已有元数据」
3. 开启「立即运行一次」
4. 查看 MoviePilot 日志：docker logs -f mpv2-moviepilot
5. 在 Emby 中抽查若干条目，确认海报/NFO 已补全且无错误覆盖
6. 关闭插件与「立即运行一次」
```

## 与其他流程的关系

```text
【正常下载】
MoviePilot 自动识别 → 自动刮削 → 自动整理 → Emby 刷新

【手动资源】
目录实时监控 → 自动整理 + 刮削 → Emby 刷新

【历史修复】（本插件）
启用 → 手动运行一次 → 补海报/NFO → 关闭

【刷流】
完全隔离，不参与刮削
```

## 注意事项

1. 不要对 `downloads/` 下任何目录做刮削。
2. 不要长期开启定时任务（如 `0 0 */7 * *`）。
3. 私享影库是否纳入刮削，按个人需求在排除路径中开关。
4. 修复完成后及时关闭插件，避免与日常整理流程重复处理。
