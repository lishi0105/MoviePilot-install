# 站点刷流插件配置

> 调度 `qB-刷流专用`，刷流资源只进 `downloads/brush`，与日常下载、媒体整理、Emby 完全隔离。

## 前置条件

- 已完成 [Readme.md](../Readme.md) §7 `init-qb`，存在下载器 `qB-刷流专用` 及分类 `brush`。
- 目录 `/volume1/media-data/downloads/brush` 已创建。
- 日常订阅下载（`qB-日常下载` → `downloads/media`）已验证正常。
- MoviePilot 插件市场已安装「站点刷流」插件。

## 配置目标

| 目标 | 说明 |
|------|------|
| 下载器隔离 | 刷流只用 `qB-刷流专用`，不用 `qB-日常下载` |
| 目录隔离 | 保存至 `downloads/brush`，不进 `media/` |
| 订阅隔离 | 不影响正常订阅下载与整理入库 |
| 风险控制 | 优先免费种、排除 H&R、保守删种 |
| 空间控制 | 按 NAS 总容量比例限制刷流占用，避免吃满磁盘 |

> 保种体积、动态删种阈值等**与容量相关的参数均按 NAS 可用总空间比例计算**，不要照搬固定 GB 数。建议先运行 2～3 天，观察上传收益、硬盘占用和插件日志后再逐步放大。

## 空间比例规划

先确认 NAS 上媒体数据盘（通常为 `/volume1/media-data` 所在卷）的**可用总容量**，再按下表比例换算为插件里的 GB 参数。

### 换算公式

```text
原始值(GB) = floor(NAS 总容量(TB) × 1024 × 比例)
参数(GB)   = floor(原始值 / 1000) × 1000
```

若已知总容量为 GB：`原始值 = floor(总容量(GB) × 比例)`，再**向下取整到整千 GB**（如 `3276` → `3000`，`6553` → `6000`）。

### 推荐比例

| 参数 | 起步（保守） | 稳定后 | 硬上限 | 说明 |
|------|-------------|--------|--------|------|
| 保种体积 | **40%** | **50%** | **60%** | 刷流做种占用目标 |
| 动态删种阈值（下限） | **40%** | **50%** | — | 删种后回落到此比例 |
| 动态删种阈值（上限） | **50%** | **62%** | **65%** | 超过此比例开始删种 |
| 留给非刷流 | ≥ **35%** | ≥ **30%** | — | 媒体库、日常下载、手动整理、系统与日志 |

**动态删种阈值**填写区间时：`下限GB-上限GB`（例如 40%～50%）。插件不支持区间时，上限填「删种触发线」那一档（起步 50%，稳定后 62%）。

### 常见容量示例

| NAS 总容量 | 保种体积（40%） | 删种阈值起步（40%-50%） | 稳定后保种（50%） | 稳定后删种（50%-62%） | 刷流硬上限（65%） |
|-----------|----------------|------------------------|------------------|----------------------|------------------|
| 8 TB | 3000 | 3000-4000 | 4000 | 4000-5000 | 5000 |
| 12 TB | 4000 | 4000-6000 | 6000 | 6000-7000 | 7000 |
| 16 TB | 6000 | 6000-8000 | 8000 | 8000-10000 | 10000 |
| 20 TB | 8000 | 8000-10000 | 10000 | 10000-12000 | 13000 |
| 24 TB | 9000 | 9000-12000 | 12000 | 12000-15000 | 15000 |

上表先按比例算出原始 GB，再 `floor(原始值/1000)×1000` 取整千，直接填入插件即可。

### 非刷流空间预留

除刷流外，至少保留 **30%～35%** 总容量给：

```text
最终媒体库（media/）
日常下载中间目录（downloads/media）
手动整理（downloads/manual）
系统缓存、数据库与日志
```

刷流长期占满会影响媒体库扩展和系统维护。

## 目录与下载器原则

| 项目 | 正确 | 错误 |
|------|------|------|
| 下载器 | `qB-刷流专用` | `qB-日常下载` |
| 保存目录 | `/volume1/media-data/downloads/brush` | `downloads/media`、`media/` |
| 种子分类 | `brush` | `media` 或其他 |

qB 中对应关系：

```text
media  → 日常订阅/搜索下载
brush  → 刷流专用
manual → 手动下载（非刷流）
private → 私密暂存（非刷流）
```

## 配置步骤

### 1. 基本配置

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 启用插件 | 开 | |
| 发送通知 | 开 | 便于观察删种、异常 |
| 立即运行一次 | 关 | 保存后手动测试一次 |
| 刷流站点 | 先选 1～2 个 | 稳定后再增加 |
| 下载器 | `qB-刷流专用` | |
| 执行周期 | `*/30 * * * *` | 每 30 分钟，不宜过频 |
| 开启时间段 | `01:00-08:00` | 夜间刷流，降低对家庭网络影响 |
| 同时下载任务数 | `5` | 保守起步，与容量无关 |
| 保种体积（GB） | **总容量 × 40%** | 见「空间比例规划」；16T 为 `6000` |
| 种子分类 | `brush` | |
| 总上传带宽（KB/s） | `5000` | 约 40Mbps，按家庭宽带上调 |
| 总下载带宽（KB/s） | `30000` | 约 240Mbps，按宽带上调 |
| 保存目录 | `/volume1/media-data/downloads/brush` | |
| 单任务上传限速（KB/s） | `1500` | 防止单种占满上行 |
| 单任务下载限速（KB/s） | `0` | 0 表示不限 |
| 自动归档记录天数 | `30` | 保留历史便于排查 |
| 动态删种阈值（GB） | **总容量 × 40%-50%** | 超上限删种，删回下限；16T 为 `6000-8000` |

若插件不支持区间写法，动态删种阈值填**上限**（起步为总容量 × 50%，16T 为 `8000`）。

### 2. 选种规则

初期**只刷免费种**，稳定后再放宽。

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 排 H&R | 是 | 必须开启 |
| 促销 | 免费 | 避免下载量风险 |
| 发布时间（分钟） | 留空 | 初期不抢新种 |
| 种子大小（GB） | `2-120` | 避免过小无收益、过大占空间 |
| 做种人数 | `3` | 过少可能慢或死种 |
| 包含规则 | 留空 | 初期不做强制包含 |
| 排除规则 | 见下方 | |

排除规则示例：

```text
成人|XXX|禁转|国语无字|日语无字|CAM|TC|TS|枪版|预告|样片|DIY|BDMV|ISO|Remux
```

**稳定后可逐步调整**（不建议一开始就激进）：

```text
促销：免费 / 2X免费
种子大小（GB）：5-200
做种人数：1
```

### 3. 删除规则

删种规则宜保守，避免误删 H&R 风险种。

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 做种时间（小时） | `72` | 普通种至少做种 3 天 |
| H&R 做种时间（小时） | `336` | H&R 风险种至少 14 天 |
| 分享率 | 留空 | 初期不按分享率删，防误删 |
| 上传量（GB） | `50` | 上传超 50GB 可作删除候选 |
| 平均上传速度（KB/s） | `20` | 长期低收益种可删 |
| 下载超时时间（小时） | `12` | 12 小时下不完可清理 |
| 未活动时间（分钟） | `4320` | 3 天无活动再删 |
| 删除排除标签 | `MOVIEPILOTV2,H&R,keep,manual` | 防误删重点任务 |

> 不建议初期配置分享率删除。部分种子未下完时上传比例可能短暂达标，导致提前被删。

### 4. 更多配置

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 站点顺序刷流 | 关 | 初期不固定站点顺序 |
| 排除订阅 | 开 | 防止影响正常订阅 |
| 动态删种子 | 开 | 达阈值后按规则控量 |
| 清除统计数据 | 关 | 保留历史便于排查 |
| 站点独立配置 | 关 | 先用全局配置 |
| 打开站点配置窗口 | 关 | |
| 删除促销过期的未完成下载 | 开 | 免费过期未下完及时清理 |
| 启用 RSS 支持 | 关 | 避免访问频率过高 |

## qB-刷流专用 配套设置

插件负责调度，qB 侧也需隔离配置（`init-qb.py` 已做基础初始化，可在 Web UI 核对）：

| 项目 | 建议值 |
|------|--------|
| 默认保存路径 | `/volume1/media-data/downloads/brush` |
| 默认分类 | `brush` |
| DHT / PeX / LSD | 关闭 |
| 最大活动下载 | `5` |
| 最大活动种子 | `200-500` |
| 最大活动任务 | `300-600` |
| 上传总限速（KB/s） | `5000-8000`（可与插件层二选一或叠加） |
| 下载总限速（KB/s） | `30000-50000` |

## 验证步骤

```text
1. 保存插件配置
2. 手动开启「立即运行一次」
3. 查看日志：docker logs -f mpv2-moviepilot
4. 确认 qB-刷流专用 新增任务，分类为 brush
5. 确认保存路径为 /volume1/media-data/downloads/brush
6. 确认没有任务进入 /downloads/media
7. 确认 Emby 未映射或扫描 brush 目录
8. 运行 2～3 天后查看上传量、硬盘占用、删种记录
9. 关闭「立即运行一次」，按观察结果决定是否长期启用
```

## 后期放大建议

运行 2～3 天稳定后，在**同一比例框架内**逐步上调（不要一次跳到硬上限）：

| 参数 | 起步 → 稳定后 | 硬上限 |
|------|---------------|--------|
| 同时下载任务数 | `5` → `8` | `10`（与容量无关） |
| 保种体积 | 总容量 × **40%** → × **50%** | 总容量 × **60%** |
| 动态删种阈值 | **40%-50%** → **50%-62%** | 刷流总占用不超过 **65%** |

示例（16T）：保种 `6000` → `8000`；删种阈值 `6000-8000` → `8000-10000`；绝对不超过 `10000` GB。

## 与其他流程的关系

```text
【日常下载】
MoviePilot → qB-日常下载 → downloads/media → 整理 → media/ → Emby

【刷流】（本插件）
站点刷流 → qB-刷流专用 → downloads/brush → 不整理、不进 Emby

【手动整理】
目录实时监控 → downloads/manual → media/ → Emby
```

## 注意事项

1. 刷流保存目录不要指向 `media/` 或 `downloads/media`。
2. Emby 不要映射 `downloads/brush`。
3. 不要让 MoviePilot 整理规则处理刷流目录（init-mpv2 已排除）。
4. 保种体积与删种阈值须按 NAS 总容量比例计算，勿照搬他人固定 GB 数。
5. 初期只刷免费种，不要关闭 H&R 排除。
6. 删种规则宜保守，分享率删除初期留空。
7. 执行周期不宜过短，避免站点访问频率过高。
8. `qB-日常下载` 与 `qB-刷流专用` 不要混用。
