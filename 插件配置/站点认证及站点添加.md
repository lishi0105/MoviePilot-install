# MoviePilot 站点认证、站点添加与 RSS 订阅

> 完成 MoviePilot 登录后：**站点认证** → **添加 PT 站点** → **填写 RSS** → **添加影视订阅**。本文以**观众**和**馒头**为例。

## 前置条件

- 已完成 [Readme.md](../Readme.md) §5 首次网页初始化，可正常登录 MoviePilot。
- NAS 可访问外网（见 Readme §1.1）。
- 已在对应 PT 站注册账号，且账号状态正常。

## 概念区分


| 步骤         | 作用                                         | 是否必须             |
| ---------- | ------------------------------------------ | ---------------- |
| **站点认证**   | 向 MoviePilot 证明你拥有某个 PT 账号，解锁搜索、订阅、站点数据等能力 | 是（至少认证 1 个支持的站点） |
| **站点添加**   | 接入 PT 站，用于搜索、订阅、下载                         | 按需               |
| **RSS 链接** | 粘贴 PT 站生成的 RSS 地址，供订阅刷新                    | 添加站点后填写          |


```text
登录 MoviePilot → 站点认证 → 添加站点 → 填 RSS → 搜索并订阅影视
  → qB-日常下载 → downloads/media → 自动整理 → Emby
```

## 一、站点认证（以观众为例）

MoviePilot V2 推荐在 Web 界面认证，无需改 `.env`。

### 1.1 获取观众认证信息

登录观众 PT 站点 Web，在控制面板查找：


| 参数          | 说明              | 常见位置                            |
| ----------- | --------------- | ------------------------------- |
| **UID**     | 用户 ID（纯数字）      | 控制面板 → 个人详情；或 RSS/种子链接中的 `uid=` |
| **Passkey** | 个人密钥（32 位左右字符串） | 控制面板 → 安全 / 隐私 → Passkey        |


示例 RSS 链接片段：

```text
...?passkey=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx&uid=12345
```

分别对应 **Passkey** 和 **UID**。

### 1.2 在 MoviePilot 中认证

```text
1. 打开 MoviePilot：http://NAS-IP:9443
2. 点击右上角用户头像
3. 选择「用户认证」
4. 在列表中找到「观众」，点击进入
5. 填写 UID、Passkey
6. 保存并等待认证结果
```

认证成功后会显示已通过；若失败，检查 UID/Passkey 是否复制完整、账号是否被站点封禁。

### 1.3 说明

- 头像菜单里**能看到的站点**即为当前版本支持的认证站；观众对应标识 `audiences`。
- 可配置多个认证站（逗号分隔时依次尝试），**任一成功即可**解锁功能。
- 认证信息与「站点添加」里的 Cookie/Token **不是同一项**；认证用 UID + Passkey，添加站点另配（见下文）。

## 二、站点添加

路径：**设定 → 站点**（或「站点管理」），点击 **添加站点**，选择目标 PT 站。

不同站点支持的鉴权方式不同，常见两类，其他选项默认即可：


| 方式                  | 适用站点示例                      | 特点                  |
| ------------------- | --------------------------- | ------------------- |
| **Token / API Key** | 馒头（M-Team）                  | 官方 API，无需 Cookie，推荐 |
| **Cookie**          | 浏览器登录态 + **User-Agent** 请求头 | 观众等传统 PT 站          |


---

## 三、Token 添加（以馒头为例）

馒头已切换为 **API 域名 + 存取令牌**，不要再用旧域名填 Cookie。

### 3.1 生成 API Key（存取令牌）

```text
1. 浏览器登录馒头 PT
2. 进入「控制台」→「实验室」（或「存取令牌」）
3. 新建存取令牌（API Key），复制保存
4. 令牌只显示一次，丢失需重新生成
```

### 3.2 获取 Authorization 请求头

除 API Key 外，馒头还需在 MoviePilot 中填写 **Authorization** 请求头：

```text
1. 保持浏览器已登录馒头
2. 按 F12 打开开发者工具 →「网络 / Network」
3. 刷新页面，或随便点开控制台、实验室等页面
4. 在请求列表中逐条查看「请求头」
5. 找到包含 Authorization 的请求，复制该字段的完整值
   （通常形如 Bearer xxxxx...）
6. 粘贴到 MoviePilot 站点配置的「请求头」中，键名为 Authorization
```

MoviePilot 请求头填写示例（按界面实际格式二选一即可）：

```text
Authorization: Bearer 你的令牌内容
```

或 JSON 形式：

```json
{"Authorization": "Bearer 你的令牌内容"}
```

> Authorization 会过期，失效后按同样步骤重新 F12 复制更新。API Key 与 Authorization **都要填**，缺一不可。

### 3.3 在 MoviePilot 添加馒头


| 字段           | 填写值                     | 说明                           |
| ------------ | ----------------------- | ---------------------------- |
| 站点           | 馒头 / M-Team             | 从内置列表选择                      |
| 站点地址         | `https://api.m-team.cc` | API 方式必须用 API 域名             |
| API Key / 令牌 | §3.1 的存取令牌              | 粘贴到 Token / API Key 字段       |
| 请求头          | `Authorization`         | 值为 §3.2 复制的 Authorization 内容 |
| Cookie       | **留空**                  | 馒头 API 模式不需要 Cookie          |
| RSS 链接       | 见 §5                    | 在 PT 站按默认选项生成后粘贴             |


### 3.4 验证

```text
1. 保存站点配置
2. 在站点列表点击「连通性测试」或「刷新数据」
3. 搜索页选择馒头，尝试搜索
4. 能出现种子列表即表示 Token 添加成功
```

> 连通测试偶发失败但搜索正常时，以能否搜到种为准。更换 Token 后需在站点管理中**手动触发一次站点修改/刷新**，以清除旧缓存。

## 四、Cookie 添加（以观众为例）

观众等传统 PT 站使用 **Cookie** 鉴权，同时还需填写 **User-Agent** 请求头（与 Cookie 同一浏览器会话获取）。推荐 **CookieCloud** 同步 Cookie，**User-Agent 需手工填写**。

### 4.1 获取 Cookie 与 User-Agent

```text
1. 浏览器登录观众 PT（与平时使用同一账号）
2. F12 →「网络 / Network」→ 刷新页面（或点开用户详情等任意页面）
3. 任选一条对观众域名的请求，在「请求头」中复制：
   - Cookie：整段 Cookie 值
   - User-Agent：整段 User-Agent 值
4. 两条须来自同一次浏览器登录，勿混用不同浏览器或设备的值
```

Cookie 也可用 CookieCloud 扩展同步；User-Agent 仍建议按上面步骤从浏览器复制。

### 4.2 在 MoviePilot 添加观众


| 字段     | 填写值             | 说明                     |
| ------ | --------------- | ---------------------- |
| 站点     | 观众              | 从内置列表选择                |
| Cookie | §4.1 复制的 Cookie | 或通过 CookieCloud 同步填入   |
| 请求头    | `User-Agent`    | 值为 §4.1 复制的 User-Agent |
| RSS 链接 | 见 §5            | 在 PT 站按默认选项生成后粘贴       |


请求头填写示例：

```text
User-Agent: Mozilla/5.0 ...
```

或 JSON 形式：

```json
{"User-Agent": "Mozilla/5.0 ..."}
```

> Cookie 与 User-Agent **都要填**，缺一不可。Cookie 过期后需重新同步或复制，User-Agent 建议一并更新。

### 4.3 方式 A：CookieCloud 同步 Cookie（推荐）

MoviePilot 内置 CookieCloud 服务（本仓库 Readme §11）：

```text
http://NAS-IP:9443/cookiecloud/
```

```text
1. MoviePilot：设定 → 系统 → 站点设置 → 开启 CookieCloud 本地服务
2. 浏览器安装 CookieCloud 扩展，填写地址、KEY、密码（见 CookieCloud 页面）
3. 登录观众 PT 后，扩展执行「同步到服务器」
4. MoviePilot 执行「同步 CookieCloud 站点」
5. 设定 → 站点 → 添加/编辑「观众」：确认 Cookie 已填入
6. 按 §4.1 复制 User-Agent 填入请求头
7. 按 §5 填写 RSS 链接 → 保存
```

### 4.4 方式 B：手工粘贴 Cookie

```text
1. 按 §4.1 复制 Cookie 与 User-Agent
2. MoviePilot → 设定 → 站点 → 添加站点 → 观众
3. 粘贴 Cookie，请求头填写 User-Agent
4. 保存并测试连通 / 搜索
```

### 4.5 验证

```text
1. 站点列表中观众显示已启用
2. 站点数据能刷新
3. 搜索页勾选观众，能返回种子结果
```

## 五、RSS 与影视订阅

### RSS 是什么、有什么用

在 MoviePilot 里添加影视订阅（想追的电影 / 剧集）后，系统需要定期从 PT 站获取**新发布的种子**。**RSS 链接**就是 PT 站提供的一条专属地址，MoviePilot 填入后会按默认间隔读取新种，自动匹配你的订阅并交给 qB 下载，再整理进 `media/`。

```text
PT 站生成 RSS 链接 → 填入 MoviePilot 站点 → 订阅影视 → 有新种自动下载整理
```

订阅相关设置**保持 MoviePilot 默认即可**。

### 生成并填写 RSS 链接

一般 PT 站点首页右上角有个像**小雷达**或**小信号 📶的按钮，点进去后选项保持默认**，点击「生成 RSS 链接」（或站点上的同类按钮），复制地址后粘贴到：

```text
MoviePilot → 设定 → 站点 → 编辑对应站点 → RSS 链接 → 保存
```

### 添加影视订阅

```text
1. MoviePilot 搜索想看的影视
2. 点击 ♥ 或「订阅」，其余保持默认
3. 匹配到新种后会下载到 downloads/media 并自动整理入库
```
