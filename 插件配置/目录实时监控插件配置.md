# 目录实时监控插件配置

> 监控 `downloads/manual`，对手动拷贝、外部导入的资源自动识别、刮削、整理入库并刷新 Emby。

## 前置条件

- 已完成 [Readme.md](../Readme.md) §9 `init-mpv2`，日常下载整理正常。
- MoviePilot 插件市场已安装「目录实时监控」插件。
- 目录 `/volume1/media-data/downloads/manual` 已由安装脚本创建。

## 适用场景

| 场景 | 说明 |
|------|------|
| 手动放入资源 | U 盘拷贝、网盘同步、手工下载后丢进监控目录 |
| 外部程序导入 | 其他工具复制文件后自动触发整理 |
| 非下载器来源 | 不走 qB 订阅/搜索，但仍需刮削入库 |
| 历史资源重整理 | 临时放入 manual 重新识别（少量、按需） |

## 不适用场景

- 日常订阅/搜索下载（由 MoviePilot + qB-media + `downloads/media` 处理）。
- 刷流保种（见 [站点刷流插件配置.md](站点刷流插件配置.md)）。
- 已入库媒体的元数据修复（见 [媒体库刮削插件配置.md](媒体库刮削插件配置.md)）。

## 目录与隔离原则

### 只监控

```text
/volume1/media-data/downloads/manual
```

### 不要监控

| 路径 | 原因 |
|------|------|
| `/volume1/media-data/media` | 已是最终媒体库，会重复整理、循环刮削 |
| `/volume1/media-data/downloads/media` | 日常下载中间目录，由 MoviePilot 自带规则处理 |
| `/volume1/media-data/downloads/brush` | 刷流目录，不应进入 Emby |
| `/volume1/media-data/downloads/private` | 私密暂存，脚本不纳入自动整理 |

## 配置步骤

### 1. 基本开关

| 选项 | 建议值 | 说明 |
|------|--------|------|
| 启用插件 | 开 | 按需启用 |
| 发送通知 | 关 | 测试阶段可临时开启 |
| 立即运行一次 | 首次测试开，之后关 | 验证配置后关闭 |
| 存储历史记录 | 开 | 避免重复处理同一文件 |
| 是否刮削 | 开 | 与 init-mpv2 转移规则一致 |
| 是否二级分类 | 开 | 匹配按地区分库的目录结构 |
| 刷新媒体库 | 开 | 整理完成后通知 Emby |
| 联动实时软连接 | 关 | 本方案使用真实文件 move |
| 联动 Strm 生成 | 关 | 不使用 strm 方案 |
| 定时任务 | 留空 | 依赖目录变更触发，无需 cron |

### 2. 监控与转移

| 选项 | 建议值 | 说明 |
|------|--------|------|
| 监控目录 | `/volume1/media-data/downloads/manual` | 仅填此一路径 |
| 监控模式 | 性能模式 | 大目录下更省资源 |
| 转移方式 | `move` | 移动至 `media/`，Emby 直接读真实文件 |
| 入库消息延迟 | `30` | 秒，批量文件时合并通知 |

**不建议**使用 `filesoftlink`，除非明确需要软链接方案。

### 3. 排除关键词

一行填写，用 `|` 或换行分隔（以插件界面为准）：

```text
@eaDir|#recycle|.recycle|.!qB|.!qb|.torrent|parts|tmp|temp|incomplete
```

用于跳过 qB 未完成文件、回收站、NAS 系统目录等。

## 验证步骤

```text
1. 保存配置，开启「立即运行一次」
2. 向 downloads/manual 放入一个已知的测试资源（小体积即可）
3. 查看 MoviePilot 日志：docker logs -f mpv2-moviepilot
4. 确认资源被识别并移动到 media/ 对应分类目录
5. 确认 Emby 媒体库已刷新
6. 关闭「立即运行一次」
```

## 与其他流程的关系

```text
【普通订阅下载】
MoviePilot → qB-media → downloads/media → 自动整理 → media/ → Emby

【手动资源整理】（本插件）
放入 downloads/manual → 目录实时监控 → 识别/刮削/整理 → media/ → Emby

【刷流】
站点刷流插件 → qB-刷流专用 → downloads/brush → 不整理、不进 Emby
```

## 推荐配置汇总

可直接对照插件界面逐项填写：

```text
【基本开关】
启用插件：开
发送通知：关
立即运行一次：首次测试开，之后关
存储历史记录：开
是否刮削：开
是否二级分类：开
刷新媒体库：开
联动实时软连接：关
联动 Strm 生成：关
定时任务：留空

【监控与转移】
监控目录：/volume1/media-data/downloads/manual
监控模式：性能模式
转移方式：move
入库消息延迟：30

【排除关键词】
@eaDir|#recycle|.recycle|.!qB|.!qb|.torrent|parts|tmp|temp|incomplete
```

## 注意事项

1. 不要监控 `media` 或 `downloads/brush`，避免重复整理。
2. 转移方式与 init-mpv2 规则一致，使用 `move` 即可。
3. 手动资源量大时，可先小批量测试，确认分类正确后再批量导入。
